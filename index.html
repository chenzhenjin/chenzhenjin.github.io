git提交代码和仓库名作域名展示index.html

git提交代码和仓库名作域名展示index.html

方式1–云端仓库下拉

先在github上new repository

在本地创建个目录 右键git bash

再进入到仓库目录

方式2–本地仓库上拉

  • 适合已有项目目录，不需要从云端拉下仓库

    先cmd到要提交代码的目录

    再使用git bash （这里是两种方式都需要执行的）

    关联仓库

提交到github

  • 这一步是会同步的云端仓库和本地仓库的代码，就是说你的本地删除的文件只要把记录提交到本地仓库，再git提交到github时就可以远程删除了

    如果你是第一次使用git工具

    主域名是你的账号

    把仓库的index.html文件通过浏览器url展示出来

  • 仓库名下的index.html

  • 浏览器通过url访问到indexhtml

    补充，如果浏览器出现项目提交信息没有显示的

  • 清除缓存,建议使用谷歌，不会出现这种问题


补充，如果要过滤上传文件

  • 要新建.gitignore，但是记事本不能生成，只能通过命令行工具(cmd或者git bash)
    touch .gitignore
  • 生成.gitignore文件并不会ls出现，要去目录看有没有生成
  • 过滤的只需每隔一行输入文件后缀名就行
  • 直接打开，过滤掉这个可恶的.project
  • 可以看到.project并不被提交上去

hexo使用教程

初始化hexo

启动hexo

新建一篇博客

打开新建的博客md文件

  • 找到source->_posts
  • 把该文件夹放在hbuilder
  • 用markdown语法书写

    放置本地图片

  • 在hexo的source目录下新建一个pic或者你喜欢的名称目录

    重新整理blog

    部署到github

  • 在new repository 的命名是账号名.github.io
  • 在hexo-blog下安装hexo-deployer-git
  • 修改_config.yml
    • 找到最后的type

  • 执行发布到github命令

    怎么查看自己的blog

    改主题

    • 浏览器url输入github.com/litten/hexo-theme-yilia观看
    • 下载主题
    • 再次修改_config.yml找到theme

      每一次提交前,之后查看时多刷新几次，不然不会生效

      致谢

    • 程序羊大佬
